Discrete Equilibrium: The System of Equations

Rendered from ~/joffrionmw.pdf — a Maple worksheet that computes the discrete-support equilibrium via Gröbner bases and real-root isolation.

Setup

Load Maple's algebraic-geometry packages and set Digits := 100. The order in which SemiAlgebraicSetTools and Groebner are loaded matters — both export a SuggestVariableOrder, and the later one wins.

restart;
with(RegularChains): with(SemiAlgebraicSetTools): Digits := 100:
with(FastArithmeticTools):
with(Groebner):

The helper \(C\) rewrites indexed names like \(x_1, p_2\) as plain symbols so that RegularChains / SemiAlgebraicSetTools can consume them (Groebner does not need this step):

C := proc (a) : subsindets(a, indexed, convert, symbol) end proc:

Primitives

\(\theta\) (also written \(t\)) is the power of the cost function; \(f_1, f_2\) are the inverse costs of valuations \(v_1, v_2\). In this worksheet they are set to \(1\), so \(v_1 = v_2 = 1\) after substitution.

\[ c(e) = e^{\theta},\qquad f_1 = 1,\qquad f_2 = 1. \] 
assume(theta, integer):
interface(showassumed = 0):
c  := proc (e) : e^theta end proc:
f1 := 1: f2 := 1:

Indifference payoffs and first-order conditions

Player 1's expected payoff at action \(x_k\) minus the indifference level \(u_1\); player 2's payoff at \(y_k\); and the marginal conditions:

\[ \begin{aligned} U_1(k) &= \sum_{j=1}^{k} q_j\,\bigl(v_1 - c(f_2 + y_j - x_k - d)\bigr) \;-\; c(x_k) \;-\; u_1, \\ U_2(k) &= \sum_{j=1}^{k-1} p_j\,\bigl(v_2 - c(f_1 + x_j - y_k + d)\bigr) \;-\; c(y_k), \\ dU_1(k) &= \frac{\partial U_1(k)}{\partial x_k},\qquad dU_2(k) = \frac{\partial U_2(k)}{\partial y_k}. \end{aligned} \] 
U1  := proc (k) : sum(q[j]*(v1 - c(f2 + y[j] - x[k] - d)), j = 1 .. k)   - c(x[k]) - u[1] end proc:
dU1 := proc (k) : simplify(diff(U1(k), x[k])) end proc:
U2  := proc (k) : sum(p[j]*(v2 - c(f1 + x[j] - y[k] + d)), j = 1 .. k-1) - c(y[k])         end proc:
dU2 := proc (k) : simplify(diff(U2(k), y[k])) end proc:

System generator sys

Let \(m\) be the number of actions of player 1 and \(n\) the number of actions of player 2 (with \(n = m\) or \(n = m+1\)). The wrapper F always calls sys with the same count for the FOCs and the indifference equations of each player, so the four formal arguments collapse to two:

  • player 1 (count \(m\)): FOCs \(dU_1(i) = 0\) and indifference \(U_1(i) = 0\) for \(i = 1, \ldots, m\);
  • player 2 (count \(n\), with \(y_1 = 0\) already pinned): FOCs \(dU_2(i) = 0\) and indifference \(U_2(i) = 0\) for \(i = 2, \ldots, n\).

Substitutions applied: \(\theta = t\), \(d = 0\), \(v_1 = f_1^t\), \(v_2 = f_2^t\), \(y_1 = 0\). (The Maple proc itself exposes the four counts n1, n2, n3, n4 for generality, but in practice \(n_1 = n_3 = m\) and \(n_2 = n_4 = n\).)

sys := proc (t, n1::integer := 0, n2::integer := 0, n3::integer := 0, n4::integer := 0) local i;
  subs( theta = t, d = 0, v1 = f1^t, v2 = f2^t, y[1] = 0,
        [ seq(dU1(i), i = 1 .. n1),
          seq(dU2(i), i = 2 .. n2),
          seq(U1(i),  i = 1 .. n3),
          seq(U2(i),  i = 2 .. n4) ] ):
end proc;

Variable ordering var

The ordering of unknowns is critical for the Gröbner / triangular-set computation. Two cases are supported: \(m = n\) and \(m + 1 = n\) (equal action counts, or player 2 has one more action than player 1):

var := proc (m::integer, n::integer) local i, r, v;
  v := []: r := [];
  for i from 2 to m do
    v := [y[i], x[i-1], op(v)]:
    r := [p[i-1], q[i-1], op(r)]:
  end do;
  if   m   = n then return [ q[m],       op(r), x[m], op(v), u[1] ]
  elif m+1 = n then return [ p[m], q[m], op(r), y[m+1], x[m], op(v), u[1] ]
  else return "Error: n = m or n = m + 1"
  end if
end proc:

System packagers F and Dev

\(F(t,m,n)\) packages the full system for solving — it calls sys with the right counts and appends the probability-closing equation (\(\sum p_i = 1\) or \(\sum q_i = 1\) depending on the case):

F := proc (t::integer, m::integer, n::integer) local i;
  if   n = m   then return C([ op(sys(t, m, m, m, m)),
                               1 - sum(q[i], i = 1 .. m) ])
  elif n = m+1 then return C([ op(sys(t, m, m+1, m, m+1)),
                               1 - sum(p[i], i = 1 .. m) ])
  else return "Error: n = m or n = m + 1"
  end if
end proc:

\(\mathrm{Dev}(t,m,n)\) returns the data used to check for a profitable deviation above the maximal bid:

  • \(\mathrm{Dev}[1]\): first-order condition for bidding above the max bid;
  • \(\mathrm{Dev}[2]\): equilibrium payoff minus the deviation payoff;
  • \(\mathrm{Dev}[3]\): the constraint that the deviation lies above the max bid.
Dev := proc (t::integer, m::integer, n::integer) local s, k;
  if   n = m   then s := [dU2(n+1), U2(n+1), y[n+1] - x[n]]:
                    s := subs(p[m] = 1 - sum(p[k], k = 1 .. m-1), s):
  elif n = m+1 then s := [dU1(n),   U1(n),   x[n]   - y[n]]:
                    s := subs(q[n] = 1 - sum(q[k], k = 1 .. n-1), s):
  else return "Error: n = m or n = m + 1"
  end if;
  return subs(theta = t, d = 0, v1 = f1^t, v2 = f2^t, y[1] = 0, s);
end proc:

Constraint sets \(N\), \(P\), \(H\)

For the cylindrical-algebraic / real-root-isolation step we declare:

  • \(N = \{u_1\}\) — variables required to be non-negative;
  • \(P(m,n)\) — variables/expressions required to be strictly positive: \(x_1\), the gaps \(x_i - y_i\) and \(y_{i+1} - x_i\), and the residual probability mass \(1 - \sum p_i\) or \(1 - \sum q_i\);
  • \(H = \emptyset\) — no inequations.
N := C([u[1]]):

P := proc (m::integer, n::integer) local i;
  if   n = m   then return C([ x[1], seq(x[i] - y[i],   i = 2 .. m),
                                     seq(y[i+1] - x[i], i = 1 .. m-1),
                                     1 - sum(p[i], i = 1 .. m-1) ])
  elif n = m+1 then return C([ x[1], seq(x[i] - y[i],   i = 2 .. m),
                                     seq(y[i+1] - x[i], i = 1 .. m),
                                     1 - sum(q[i], i = 1 .. m) ])
  else "Error n = m or n = m + 1"
  end if
end proc:

H := []:

Main driver SOL

For each \(1 \le i \le k\) and \(j \in \{i, i+1\}\):

  1. Compute a Gröbner basis of \(F(t, i, j)\) in plex order.
  2. Eliminate solutions inherited from lower-order systems (divide out by the previous basis polynomial).
  3. Isolate real roots with RealRootIsolate (Discoverer method).
  4. Run the deviation test using \(\mathrm{Dev}(t,i,j)\); report whether a profitable deviation exists.
SOL := proc (k::integer, t::integer) local i, j, B0, C0, B, R, S, B1, devt, Bd, Nd, Pd, Rd, Sd;
  B1 := []: B0 := 1: C0 := 1:
  for i from 1 to k do
    for j from i to i+1 do
      R := PolynomialRing(C(var(i, j))):
      B := Basis(F(t, i, j), plex(op(C(var(i, j)))), method = convert):
      if i = j then B1 := [ simplify(B[1]/B0), op(B[2 .. nops(B)]) ]: B0 := B[1]:
               else B1 := [ simplify(B[1]/C0), op(B[2 .. nops(B)]) ]: C0 := B[1]:
      end if:
      S := RealRootIsolate(B1, N, P(i, j), H, R, method = 'Discoverer');
      if S = [] then printf("No solution for case %a\n", [i, j]);
      else
        print(Display(S, R));
        devt := Dev(t, i, j):
        Bd := [ op(B1),       C(devt[1]) ]:
        Nd := [ op(N),        C(devt[2]) ]:
        Pd := [ op(P(i, j)),  C(devt[3]) ]:
        if i = j then Rd := PolynomialRing([C(y[i+1]), op(C(var(i, j)))]):
                 else Rd := PolynomialRing([C(x[j]),   op(C(var(i, j)))]):
        end if:
        Sd := RealRootIsolate(Bd, Nd, Pd, H, Rd, method = 'Discoverer'):
        if Sd <> [] then printf("Profitable deviation in case %a !!!\n", [i, j]);
                          print(Display(Sd, Rd));
                     else printf("No profitable deviation for case %a.\n", [i, j]);
        end if:
      end if:
    end do
  end do
end proc:
