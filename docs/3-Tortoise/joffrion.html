Maple code to numerically compute the equilibrium candidate and evaluate its payoff.

Setup

Restart Maple, declare \(\theta\) as an integer parameter, and set working precision to 100 digits (crucial for the numerical solver).

restart; assume(theta, integer); interface(showassumed = 0);
Digits := 100;

Primitives

CDF of the continuous, symmetric case, the cost function, and its inverse:

\[ G(x,v) \;=\; e^{x/\sqrt{v}} - 1, \qquad c(e) \;=\; e^{\theta}, \qquad f(c) \;=\; c^{1/\theta}. \] 
G := proc (x, v) exp(x/sqrt(v)) - 1 end proc;
c := proc (e) e^theta end proc;
f := proc (c) c^(1/theta) end proc;

Players' utilities and marginal utilities

Player 1's expected utility when choosing the \(k\)-th action level \(x_k\), and its derivative with respect to \(x_k\):

\[ U_1(k) \;=\; \sum_{j=1}^{k} q_j\,\Bigl(v_1 - c\bigl(f(v_2) + y_j - x_k - d\bigr)\Bigr) \;-\; c(x_k), \qquad dU_1(k) \;=\; \frac{\partial U_1(k)}{\partial x_k}. \]

Player 2's expected utility at the \(k\)-th action \(y_k\), and its derivative:

\[ U_2(k) \;=\; \sum_{j=1}^{k-1} p_j\,\Bigl(v_2 - c\bigl(f(v_1) + x_j - y_k + d\bigr)\Bigr) \;-\; c(y_k), \qquad dU_2(k) \;=\; \frac{\partial U_2(k)}{\partial y_k}. \] 
U1    := proc (k) : sum(q[j]*(v1 - c(f(v2) + y[j] - x[k] - d)), j = 1 .. k)     - c(x[k]) end proc;
dU1   := proc (k) : simplify(diff(U1(k), x[k])) end proc;
U2    := proc (k) : sum(p[j]*(v2 - c(f(v1) + x[j] - y[k] + d)), j = 1 .. k-1)   - c(y[k]) end proc;
dU2   := proc (k) : simplify(diff(U2(k), y[k])) end proc;

Equilibrium equations

Indifference / first-order conditions packaged as sequences:

\[ \begin{aligned} \mathrm{eqU}_1(n) &= \bigl(\,U_1(k) - U_1(1)\,\bigr)_{k=2,\dots,n}, \\ \mathrm{eqU}_2(n) &= \bigl(\,U_2(k)\,\bigr)_{k=2,\dots,n}, \\ \mathrm{eqdU}_1(n) &= \bigl(\,dU_1(k)\,\bigr)_{k=1,\dots,n}, \\ \mathrm{eqdU}_2(n) &= \bigl(\,dU_2(k)\,\bigr)_{k=2,\dots,n}. \end{aligned} \] 
eqU1  := proc (n) : seq(U1(k) - U1(1), k = 2 .. n) end proc;
eqU2  := proc (n) : seq(U2(k),         k = 2 .. n) end proc;
eqdU1 := proc (n) : seq(dU1(k),        k = 1 .. n) end proc;
eqdU2 := proc (n) : seq(dU2(k),        k = 2 .. n) end proc;

Full system and monotonicity constraints

  • aux(n) — the basic equations of the system: \(\{y_1\} \cup \mathrm{eqU}_1(n) \cup \mathrm{eqU}_2(n) \cup \mathrm{eqdU}_1(n) \cup \mathrm{eqdU}_2(n)\).
  • ord(n) — monotonicity: \(x_k < x_{k+1}\) and \(y_k < y_{k+1}\) for all \(k\).
  • sys(n,t) — substitutes \(\theta = t\) and the closing condition \(q_n = 1 - \sum_{k=1}^{n-1} q_k\) into \(\mathrm{aux} \cup \mathrm{ord}\).
  • nsys(n,t) — same closing condition, but without the inequality constraints (the numerical solver only takes equations).
aux := proc (n) options operator, arrow;
  {y[1], eqU1(n), eqU2(n), eqdU1(n), eqdU2(n)}
end proc;

ord := proc (n):
  `union`( {seq(x[k] < x[k+1], k = 1 .. n-1)},
           {seq(y[k] < y[k+1], k = 1 .. n-1)} )
end proc;

sys  := proc (n, t) :
  subs(theta = t, q[n] = 1 - sum(q[k], k = 1 .. n-1),
       `union`(aux(n), ord(n)))
end proc;

nsys := proc (n, t) :
  subs(theta = t, q[n] = 1 - sum(q[k], k = 1 .. n-1), aux(n))
end proc;

Initial guess for the numerical solver

Effort levels start on a uniform grid on \(\bigl[0,\,\ln(2)\sqrt{v}\,\bigr]\); probabilities are evaluated using the CDF of the symmetric quadratic-case equilibrium:

\[ p_k = G\!\left(\tfrac{k\,\ln 2\,\sqrt{v}}{n},\,v\right),\quad q_k = G\!\left(\tfrac{(k-1)\,\ln 2\,\sqrt{v}}{n},\,v\right),\quad x_k = \tfrac{k\,\ln 2\,\sqrt{v}}{n},\quad y_k = \tfrac{(k-1)\,\ln 2\,\sqrt{v}}{n}. \] 
vars := proc (n, v) options operator, arrow;
  evalf({
    seq(p[k] = G(k*ln(2)*sqrt(v)/n,   v), k = 1 .. n-1),
    seq(q[k] = G((k-1)*ln(2)*sqrt(v)/n, v), k = 1 .. n-1),
    seq(x[k] = k*ln(2)*sqrt(v)/n,           k = 1 .. n),
    seq(y[k] = (k-1)*ln(2)*sqrt(v)/n,       k = 1 .. n)
  })
end proc;

Solution helpers

spit strips the right-hand sides out of a Maple solution list; param repackages them into the four lists \([p],\,[q],\,[x],\,[y]\).

spit := proc (a)::list;
  local k, i, s;
  description "gets the solution of an equation and returns only the values";
  s := NULL; k := nops(a);
  for i to k do s := [s[], rhs(op(i, a))] end do;
  s
end proc;

param := proc (a)::list;
  local k, i, j, p, q, x, y, b;
  description "gets the solution of our system and returns only the values";
  k := (1/4)*nops(a) + 1/2;
  b := spit(a);
  p := NULL; q := NULL; x := NULL; y := NULL;
  for i to k-1                do p := p, op(i, b) end do;
  p := [p, 1 - sum(p[j], j = 1 .. k-1)];
  for i from k    to 2*k-2    do q := q, op(i, b) end do;
  q := [q, 1 - sum(q[j], j = 1 .. k-1)];
  for i from 2*k-1 to 3*k-2   do x := x, op(i, b) end do;
  for i from 3*k-1 to 4*k-2   do y := y, op(i, b) end do;
  return [p, q, [x], [y]];
end proc;

Worked example — quadratic symmetric case, two actions each

Set \(v_1 = v_2 = 1\), \(d = 0\), \(n = 2\), \(\theta = 3\), and call fsolve:

S0    := subs(v1 = 1, v2 = 1, d = 0, nsys(2, 3));
vars0 := subs(v = 1, vars(2, 3));
sol   := fsolve(S0, vars0);
spit(sol);

Payoffs as plotting-friendly functions

Re-express \(U_1, U_2\) as ordinary functions of one variable (for plotting), and build the piecewise payoffs \(\mathrm{PEU}_1, \mathrm{PEU}_2\) that switch branch at the thresholds \(y_{k+1}\) and \(x_k + a\) respectively, where \(a = d + f(v_1) - f(v_2)\):

EU2 := unapply(subs(y[k] = y, theta = t, U2(k)),
               y, t, k, p, x, v1, v2, d);
EU1 := unapply(subs(x[k] = x, theta = t, U1(k)),
               x, t, k, q, y, v1, v2, d);

PEU1 := proc (p, q, x, y, t, n, v1, v2, d, z)
  local a; a := d + subs(theta = t, f(v1) - f(v2));
  subs(q[n] = 1 - sum(q[k], k = 1 .. n-1),
    piecewise(
      seq([z + a < y[k+1], EU1(z, t, k, q, y, v1, v2, d)][], k = 1 .. n-1),
      EU1(z, t, n, q, y, v1, v2, d)))
end proc;

PEU2 := proc (p, q, x, y, t, n, v1, v2, d, z)
  local a; a := d + subs(theta = t, f(v1) - f(v2));
  subs(p[n] = 1 - sum(p[k], k = 1 .. n-1),
    piecewise(
      seq([z < x[k] + a, EU2(z, t, k, p, x, v1, v2, d)][], k = 1 .. n),
      EU2(z, t, n+1, p, x, v1, v2, d)))
end proc;
psol := param(sol); nops([psol]);

Consistency check

Verifies that the numeric solution respects the monotonicity constraints: returns "OK!" if \(y_1 \ge x_n + a_1\) and \(x_1 \ge y_n - a_2\), where \(a_1 = d - f(v_2)\) and \(a_2 = d + f(v_1)\); otherwise "ERROR!".

Cst := proc (p, q, x, y, t, n, v1, v2, d)
  local a1, a2;
  a1 := d - subs(theta = t, f(v2));
  a2 := d + subs(theta = t, f(v1));
  if y[1] < x[n] + a1 or x[1] < y[n] - a2
    then "ERROR!"
    else "OK!"
  end if
end proc;

Cst(psol, 3, 2, 1, 1, 0);
