Can we borrow anything from Rentschler & Turocy (2015) for the elimination contest?

Internal note · 2-ElimContest · June 17, 2026. Scope: theory only (deep-learning / numerical solver ignored, per request). Question: do the bid-support tools in Turocy's algorithm transfer to our two-round IPV elimination contest? Short answer below, reasoning after.

Bottom line. The IPV-vs-common-value contrast is not the obstacle — Turocy's framework already nests private values as a special case, so it applies verbatim to our Round 2 subgame. The real obstacle is that our game is two-stage with an endogenous, bid-dependent continuation value, which Turocy's single-shot machinery does not model. The genuinely transferable pieces are (i) the admissible-active-set feasibility test as a support-pruning tool, and (ii) the piecewise-uniform / finite-disjoint-interval structural predictions, which can be used as a fixed-point inner module and as a validation target for the numerical binary solver. No eureka, but two concrete, usable hooks.

1. The two models, side by side

The decisive observation: Turocy's primitive is the matrix $\psi_{l|k}\equiv h_{l|k}\,V_{k,l}$, where $h_{l|k}$ is the belief that the rival is type $t_l$ given own type $t_k$, and $V_{k,l}$ is the value of winning. Pure IPV is just the special case $V_{k,l}=v_k$ (value independent of rival), with $h$ the prior. So "they are common value, we are IPV" is a non-issue: their construction was built to span both. What matters is whether the monotonicity condition KMS ("$\psi_{l|k}$ strictly increasing in $k$ for every $l$") holds — and in our non-separating binary case it precisely fails, which is exactly the regime their algorithm targets.

2. The Turocy machinery, in one screen

Expected payoff to bidding $b$ as type $t_k$ against rival strategy $\Pi$:

$$ u_k(b\mid\Pi)=\sum_{l:t_l\in T} \psi_{l|k}\Big[\Pi_l\big([0,b)\big)+\tfrac12\Pi_l(\{b\})\Big]-b .$$

The construction characterizes where bids pile up:

  • Active set $A$ on an interval $(e,e{+}L)$: the types indifferent (best-responding) there.
  • Admissible active set (Def. 3): $A$ for which the linear system $$\sum_{l:t_l\in T}\psi_{l|k}\,\mu_l = L \quad \forall k:t_k\in A,\qquad \mu_k\ge 0,\ \mu_k=0\ \forall t_k\notin A,\ L\ge 0$$ has a solution. This is a Farkas / linear-feasibility test — it decides, without ever fixing the bid level, which subsets of types are allowed to randomize on a common bid interval.
  • Partial order $\succ$ on admissible sets $\to$ directed graph $\Gamma$; any equilibrium is a path through $\Gamma$. Boundaries between intervals = active-set changes.
  • Gap subpaths (Def. 6): a type active, then inactive, then active again $\Rightarrow$ its support has disjoint components. Non-monotone bidding lives here.
  • Feasibility of a whole path = linear system (5)–(6) (mass conservation $\sum_A\mu_k^A=1$ + zero net-payoff over each gap).

Structural guarantees worth keeping:

P4 / P7. There always exists an equilibrium with piecewise-uniform randomization, and each type randomizes over no more than $|\mathcal X_k|+1$ disjoint intervals (finite). Also $\inf E(\Pi)=0$: the bottom of the overall support is $0$, so the lowest-value active type's support starts at $0$.

3. What transfers — item by item

USE Round 2 is a Turocy instance

After full revelation, the final (winner-A vs winner-B) all-pay subgame is literally a two-bidder all-pay auction with discrete types and interdependent beliefs: types = finalists' realized values, beliefs $h_{l|k}$ = posteriors inferred from the observed Round-1 bids, $V_{k,l}=v_k$ (IPV). So Turocy's admissibility test and support characterization apply directly to Round 2.

Caveat / honesty: the README says Round 2 is already solved in closed form (Q-function, Lu–Parreiras; closed form in the binary case). So here Turocy mostly confirms structure rather than adding firepower — useful as an independent cross-check, not a new result.

ADAPT Admissibility test as a Round-1 support-pruner

For the binary case the candidate active sets are only $\{L\},\{H\},\{L,H\}$. The admissibility system for the co-randomizing set $\{L,H\}$ on an interval is just two equations:

$$\psi_{L|L}\mu_L+\psi_{H|L}\mu_H=L,\qquad \psi_{L|H}\mu_L+\psi_{H|H}\mu_H=L,\qquad \mu_L,\mu_H\ge0 .$$

Solving this (and the singleton cases) tells you, before guessing bid levels, whether both types can share a bid interval and in what mass ratio — exactly the "where do the bid supports sit / do they overlap" question. The "hide low, bluff high" non-monotonicity should show up as a gap subpath for the high type (active low, inactive in the middle, active high) and/or an active-set ordering that reverses the naive value order.

The obstruction (be explicit). In Round 1 the entries $\psi$ are not constants. The "value of winning Round 1" is the continuation value $Q$, which is endogenous: it depends on the rival's full Round-1 bid distribution and on the off-path beliefs your bid induces in the finalists you will face. So $\psi^{R1}_{l|k}=\psi^{R1}_{l|k}(b)$ — bid-dependent — whereas Turocy's whole linear-algebra edifice requires fixed $\psi$. The admissibility test does not drop in unchanged for Round 1.

ADAPT Turocy as the inner module of a fixed point

This is the most promising idea. Wrap the static construction in an outer continuation-value iteration:

  1. Fix a candidate Round-2 continuation-value function $Q(\cdot)$ (from the closed-form binary R2 solver).
  2. On a bid grid, linearize Round-1 payoffs into an effective, locally-constant $\psi^{R1}$ matrix per interval, treating $Q$ as the value of advancing.
  3. Run Turocy's admissible-set enumeration to get candidate Round-1 support structures (which intervals, which active sets, gap subpaths).
  4. Recompute beliefs and $Q$ from the resulting bid distribution; iterate to a fixed point.

This uses Turocy's combinatorial support enumeration as a structure generator inside the loop our project already runs numerically — potentially giving an analytic skeleton (interval count, active-set sequence) that the deep-learning solver currently only discovers empirically.

USE Structural predictions as validation targets

Even with no algorithmic import, P4/P7 give falsifiable predictions for the binary Round-1 equilibrium that binary_report.py can check directly:
  • Each type's support = finite union of intervals, piecewise-uniform density (not smooth/atomic).
  • Binary $\Rightarrow$ at most 2 disjoint intervals per type ($|\mathcal X_k|\le1$ with two types).
  • Lowest bid in the union of supports is exactly 0.
  • Non-monotone bidding manifests as interleaved/overlapping active-set intervals with a gap for one type — a sharp signature to look for in the histograms, distinct from mere noise.

SKIP Direct reuse of the code & the path enumerator

The companion enumerator.py (sympy, networkx, Python 2.7) solves the static game by enumerating paths through $\Gamma$. It cannot be pointed at our two-stage game as-is: it has no notion of a revelation stage or an endogenous, bid-dependent $V$. Cycles in $\Gamma$ (their Example 5) also signal that brute-force path enumeration is fragile — irrelevant for binary (only singletons + one pair) but a warning if we ever push to $K\ge3$ analytic types. Reuse the idea (admissibility + paths), not the code.

4. Selection — a side benefit

Turocy's Examples 3a/3b dwell on multiplicity and selection when KMS fails (multiple equilibrium supports; a "bid-0-in-everyone's-support" degenerate selection vs. a separating one). Our project already imposes an off-path belief refinement ($\mu_h=1$ above the iterated-dominance cap) to pin down the equilibrium. Turocy's discussion is a useful external reference point: it confirms that without a refinement, non-monotone all-pay games generically have multiple equilibria with different supports and payoffs — i.e. our $\mu=1$ device is doing necessary work, not cosmetic.

Honest summary, as requested: no breakthrough. Turocy does not solve the elimination contest, because it is single-stage and our hard part (Round 1) has an endogenous, bid-dependent value of advancing. But two things are worth carrying forward — (1) its support-structure theorems as concrete predictions to test the binary Round-1 numerics against, and (2) its admissibility/Farkas test as a candidate inner module in a continuation-value fixed-point. The common-value vs IPV distinction, which prompted the worry, turns out to be the one thing that is not a problem.
